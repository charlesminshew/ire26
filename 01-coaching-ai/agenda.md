# Agenda — Coaching AI Tools for Investigative Reporting

**IRE 2026 · Friday, June 26 · 2:45 p.m. · Maryland 3-4 · National Harbor, Maryland**

Charles Minshew, The Atlanta Journal-Constitution

| Time | Block | What happens |
|------|-------|--------------|
| 0:00–0:05 | Intro | Ground rules: what AI is good for, what it isn't. The "argue with it" mindset. |
| 0:05–0:15 | Demo + exercise | **Public records requests** — bad prompt vs. coached prompt; attendees draft a FOIA for something they're working on |
| 0:15–0:27 | Demo + exercise | **Understanding data** — paste a field list or sample rows; get AI to explain the dataset, flag problems, identify scope |
| 0:27–0:38 | Demo + exercise | **Reporting plan from a dataset** — using the same data, ask AI to generate story questions and a step-by-step analysis plan |
| 0:38–0:48 | Demo + exercise | **Spreadsheet tasks** — bring a formula or analysis problem; AI writes and explains it |
| 0:48–0:54 | Demo + exercise | **Debugging scripts** — structured debugging prompt; try on a broken snippet or their own code |
| 0:54–0:57 | Patterns to take home | Point to tipsheet, 2–3 reusable prompt patterns to try first |
| 0:57–1:00 | Q&A + wrap | Questions, repo URL |

## Notes

- Debugging is the most optional block for a general IRE audience — cut it if earlier exercises run long.
- The data understanding and reporting plan blocks work best with the same dataset so attendees see the full arc from "what is this?" to "what's the story?"
