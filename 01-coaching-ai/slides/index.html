What we'll do today

Five hands-on moves — built around one records request and one real dataset. First, a quick reality check on where AI helps and where it confidently lies.

  • 1️⃣ Coach a FOIA request — get specific, iterate, ask for the data
  • 2️⃣ Make sense of an unfamiliar dataset — data dictionary + scope
  • 3️⃣ Turn that same data into a reporting plan — questions + steps
  • 4️⃣ Get Excel help — formulas explained, no code required
  • 5️⃣ Debug a broken script — even if you don't write code

What you'll need

  • ✅ A free ChatGPT or Claude account
  • ✅ A laptop (preferably no tablets)
  • ✅ An interest in data and open records
  • ✅ The willingness to argue with a generative AI service

Do use AI for…

Writing & debugging code
Understanding unfamiliar data
Internal documentation
Drafting public records requests
Taking better notes for yourself
A super-powered search engine
//

Don't use AI for…

Final data analysis work
Handling sensitive personal data
Replacing the work of teammates
Processing entire datasets
Writing stories for the public
Making editorial decisions
//

These guidelines are just suggestions. Consult with your bosses or attorney for more specific recommendations.

This is very important!

Where AI will confidently lie to you

  • ⚠️ Made-up citations, case numbers and statutes
  • ⚠️ Specific facts, dates and figures it never actually had
  • ⚠️ "Yes, I checked" — when it checked nothing
  • ⚠️ Math and totals on data you pasted in

Rule of thumb: AI is great at structure and language, risky on facts. Verify everything you'd verify from a source.

FOIA: start specific

◆ Be specific
Write a records request for elevator inspection reports at 10 S West St, Indianapolis, IN.
◆ Be more specific
You're a reporter and you need information about elevator inspections at the JW Marriott at 10 S West St, Indianapolis, IN. Write a records request seeking reports from the Indiana Department of Homeland Security.

FOIA: iterate toward the data

◆ Iterate
You're a reporter and you need to put your elevator reports from the JW Marriott into context. Write a records request to the Indiana Dept. of Homeland Security for inspection data.
◆ Iterate some more
Write an open records request to the Indiana Dept. of Homeland Security for a state database of elevator inspection reports since January 1, 2020 — including all releasable fields.

Your turn — pick a beat

I'm a reporter covering [housing / schools / police / courts]. Write a public records request to [agency] seeking data about [issue]. Ask for an export with all releasable fields, then list the fields I should request.

Stuck? Try: building inspections · overtime pay · school discipline · 911 response times

Understand the data

◆ Data dictionary
Here are the column names from a Prince George's County residential & commercial permits dataset. Write a plain-English data dictionary. [paste fields]
◆ Scope & limits
Based on these fields, what does this dataset actually capture — and what does it miss? What should a journalist know before relying on it?

Build a reporting plan

Prompt What are five investigative story ideas this permit data could support? For each one, list the key questions to answer and the specific analysis steps to get there.
Push back Which of these is most realistic with only the fields we have? What additional records or data would I need to FOIA to finish the strongest story?

Excel without writing code

Ask for the formula I'm in Excel. Column C has permit issue dates and column F has permit type. Write a formula that counts how many "Residential" permits were issued in 2024. Then explain each part in one sentence.
When it breaks That formula returns #VALUE!. Here's exactly what I typed: [paste]. What's wrong and what's the minimal fix?

When a colleague's script breaks

You don't have to write code to get unstuck.

The debugging pattern Here's a script a colleague wrote: [paste code]. Here's the error: [paste error]. Don't rewrite the whole thing. Tell me: (1) what's causing the error, (2) the minimal change to fix it, (3) anything else obviously wrong.

Three things to try Monday

  • 1️⃣ Draft your next records request with AI — then iterate it toward data, not just documents.
  • 2️⃣ Paste an unfamiliar dataset's columns in and ask for a data dictionary + scope.
  • 3️⃣ Next time a formula or script breaks, paste the error and ask for the minimal fix.

Always verify facts. AI is your intern, not your editor.
