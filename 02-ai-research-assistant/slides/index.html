Five tasks that eat your time

AI does more than write emails and summarize. Today we put it to work on real reporting chores.

  • 1️⃣ Pull data off a web page, no scraper
  • 2️⃣ Turn a messy PDF into a structured spreadsheet
  • 3️⃣ Compare two versions of a bill to find what changed
  • 4️⃣ Clean and standardize messy names or addresses
  • 5️⃣ Build a quick app to browse and search your documents

Trust it to transform, not to know

AI is great at transforming: reshaping, extracting, comparing.
It is risky on facts.

Spot-check everything it pulls against the source before you trust it.

Three ways to hand AI your material

  • 📎 Upload a file: the paperclip or “+” button. PDFs, spreadsheets, images.
  • 📋 Paste the text: a column of values, a page's content, a code snippet.
  • 🔗 Give a URL: if your tool can browse, paste the link and let it read.

Tip: tell it what the thing is and what you want back (a table, CSV, a diff). Specifics beat politeness.

1 · Web page → table

Sample: flycou.com passenger load data · one page per year (2002–2025)
◆ Extract
Here's a web page of monthly airport passenger numbers: [paste URL or page text]. Extract every row into a table with columns: month, enplanements, deplanements, monthly total, % change vs. prior year. One row per month. If a value is missing, leave it blank; don't guess. Return as CSV.
◆ Then scale it
That's only 2025. Here's the 2024 page; append those rows in the same format, with a "year" column. (Repeat back to 2002.)

Every airport, every checkpoint, every hour. Nobody copy-pastes this by hand.

Source: tsa.gov/foia/readingroom

2 · Messy PDF → spreadsheet

Sample: most recent weekly TSA throughput report (PDF, ~1,000 pages)
◆ Extract
This PDF is a weekly TSA throughput report. Extract the data into a table with these columns: [list them]. Don't infer values that aren't in the document. Dates as YYYY-MM-DD; numbers plain (no $ or commas). Keep every page. Return as CSV.
◆ Validate
List any rows where a value looked ambiguous or hard to read, and tell me which page each came from, so I can check the original.

3 · Compare two versions

Sample: GA HB 268 school-safety bill, as introduced vs. as passed
◆ Diff
Here are two versions of a bill: VERSION A (as introduced) and VERSION B (as passed). Show everything that changed A→B: language added, removed, and reworded (show before/after). Group by section. Ignore formatting-only differences. Flag any change that shifts who benefits, who pays, deadlines, or penalties.

Watch for the buried shift: this bill drifted from mental-health provisions toward physical security & surveillance.

4 · Clean messy names & addresses

Sample: Fulton County precinct list, inconsistent names & addresses
◆ Standardize
Here's a column of [precinct locations / addresses] entered inconsistently by different people. Make a table with two columns: the original value, and a standardized version. Standardize capitalization and abbreviations (St/Street, Inc/Incorporated). Do NOT merge two values unless you're confident they're the same entity. Flag any you're unsure about in a third column.

The “don't merge” flag is the point: near-duplicates aren't always the same place.

5 · Build an app by iterating

Sample: 9,091 Habersham County ballots: precinct, party, governor vote (CSV)
◆ Start small
I have a CSV of ballot records. Write one self-contained HTML file that shows them in a searchable, sortable table. No build tools, no server; it runs by opening the file. Put the data at the top so I can swap in my own.
◆ Then keep asking
Now add a filter by precinct and by party, and a count of matching rows. → Then: let me click a row for the full record. → Then: the governor votes have OCR typos; group near-identical names so I can spot them.

The skill isn't the first prompt. It's the back-and-forth: ship something tiny, look at it, ask for the next thing.

The deeper version: 162,847 ballots

Cobb County scanned ballot images → structured data on a governor's race and two Supreme Court races. Same pattern as today, at scale:

  • ① Find the machine-readable layer (a clean text summary page)
  • ② Write code to extract: OCR + regex across 162K files
  • ③ Exceptions are a second dataset: build review tools (see a live one)
  • ④ Verify totals against official results before you publish

AI accelerated the coding, not the journalism. The judgment stayed with the reporter.

Three things to try Monday

  • 1️⃣ Next messy PDF → ask for a CSV, then ask which rows to double-check.
  • 2️⃣ Two versions of anything → ask for a grouped diff and what shifts who benefits.
  • 3️⃣ A pile of records → ask for a one-file app to search and browse them.

Always verify facts. AI is your assistant, not your editor.
